How it works

Four agents.
One goal.

Each prompt is decomposed by an orchestrator and routed to the right specialist. They collaborate in parallel, validate each other, and ship a coherent project — not a pile of fragments.

01 · Orchestrator

Decomposes the prompt.
Assigns the work.

Reads your one-liner, drafts a plan, breaks it into tasks, and dispatches each task to the right agent with clear acceptance criteria.

02 · Designer

Sketches the product.
Picks the system.

Defines the visual language, screens and content. Ships a design brief that downstream agents can implement without guessing.

03 · Frontend

Builds what users see.
Every pixel on purpose.

Implements screens from the brief with real components, real interactions and a real build — not a static mock.

04 · Backend

Wires the logic.
Verified end-to-end.

Sets up APIs, data models and integrations, then runs automated checks against the orchestrator's acceptance criteria.

From sentence to system

Watch a prompt
become a product.

One line in. A full-stack project out. The same flow you saw above, rendered live — no shortcuts, no mocked screens.

opc run "build me a job board for designers"
  • Real components, not lorem

    Every screen ships with typed, reviewable code — not a Figma snapshot.

  • Real APIs, real data shape

    Backend and frontend agree on the contract before a single line of UI is rendered.

  • Real checks, real failures

    If a step fails, OPC loops the responsible agent — and tells you when it can't recover.

  • Real artifact, on disk

    You get a runnable project at ./out/your-project/, not a chat transcript.
